- [ ] T-000002: Behavioral-target videos are re-extracted, re-Vision-filtered, and re-described on every subsequent run: when `spec and spec_sources`, `_run_video_frame_pipeline(behavioral_paths)` (pipeline.py ~1723-1736) runs unconditionally with no change gating, and if the video was also among the changed ref/ files it was already processed by `_analyze_new_files` earlier in the same run (~925) -- double Vision/LLM spend in a single invocation, plus duplicate `store_accepted_frames`. - [ ] T-000003: A real build task phrased "Verify ..." is misclassified as verification and then auto-deleted: `_VERIFY_PREFIX_RE` (plan_sanity.py ~46) treats any task starting with verify/test-that as verification, so its `[feat]` never enters `built_features`; `_check_verification_mapping` then flags KIND_VERIFY_WITHOUT_BUILD and `_repair_verify_without_build` (plan_gate.py ~103-124) removes the line -- silently deleting genuine feature work from PLAN.md.
