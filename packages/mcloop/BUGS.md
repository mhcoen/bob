## Bugs
- [ ] Formatting discipline varies by editor model: the codex/gpt-5.5 fallover tier ships unformatted code that Claude-family editors would have formatted, so a model swap mid-phase changes the artifact quality. Observed 2026-06-11 (writer, task 20.5): fable failed 3 attempts, the chain fell over kimi/deepseek (402) to codex/gpt-5.5, which completed the task with a test file that failed the phase boundary's `ruff format --check` -- the task-level acceptance (pytest) passed, so the defect surfaced only at the boundary, stopping the run a phase late and requiring a hand fix. Formatting must not depend on which model held the editor seat. Fix: run the project's formatter deterministically in the loop after every editor session and before the acceptance check (e.g. `ruff format` on the changed files when the project configures ruff), the same way commits are mcloop's job and not the model's -- format-on-exit is scaffolding work, not a model habit to hope for. Regression test: a stubbed editor session that writes a deliberately misformatted file still yields a task commit that passes `ruff format --check`.
