## Bugs

- [x] T-000001: `run_workflow` leaks the SQLite ArtifactStore on any executor/summary exception: api/dispatch.py ~256 opens the store but `store.close()` runs only on the success paths (~320/~325); the try/finally at ~300-307 closes the log, not the store, so each errored run in a long-lived REPL/server process leaks a connection/fd until exhaustion. Same shape in `cmd_resume` (cli.py ~427-531): the store and LogWriter are opened before the try/finally that closes them, so a failure in run_resume_hooks, VisibilityIndex setup, or Executor construction leaks both. Wrap the store and log lifetimes in try/finally from the point of open in both places. Regression test: a raising executor leaves no open store connection. - [x] T-000002: One invalid UTF-8 byte truncates all captured session output: run_session opens the child's stdout in text mode with strict decoding (adapters/_subprocess.py ~808-817, no errors= argument), and the reader thread swallows the resulting UnicodeDecodeError (~859) then queues the sentinel, so capture ends early and the final result record is lost while exit_code can still be 0. `extract_final_text` then returns an empty or partial answer. Use errors="replace" on the stream. Regression test: a child emitting a bad byte mid-stream still yields the trailing result record. <!-- completed_at: 2026-07-04T02:09:32Z -->
- [x] T-000003: Idle-kill and interrupt exit codes are classified as hard errors instead of timeouts: `_verdict_for_exit_code` in all four real adapters (e.g. claude_code_text.py ~329-342) maps only -2 to timeout, but run_session kills a stuck session with -3 after IDLE_TIMEOUT_S (adapters/_subprocess.py ~908) -- a genuine timeout reported as error, contradicting the adapter docstrings and potentially driving different retry behavior upstream. Map -3 to timeout in all four adapters and make a deliberate decision about 130 (interrupt). Regression test per adapter. <!-- completed_at: 2026-07-04T02:12:54Z -->
